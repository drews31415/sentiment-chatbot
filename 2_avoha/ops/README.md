WoZ 운영 콘솔 + 시드/동기화/export 스크립트 + 운영 SOP.

## 운영 콘솔 (WoZ)
- 배포: `console.avoha.today` (Railway 또는 Vercel, 동일 백엔드)
- 인증: Google SSO + `OPS_ALLOWED_EMAILS` 화이트리스트
- 핵심 UX: 키보드만으로 처리 가능 (`1` 승인, `2` 수정, `3` 재질문)

## 인접 파트 인터페이스
- **Backend API**: `/ops/*` 네임스페이스 소비 (세션 쿠키)
- **Backend DB**: `scripts/*` 는 `DATABASE_URL` 직접 접속 (읽기·시드 전용)
- **Frontend**: 동일 백엔드, 별도 서브도메인
- **Design**: 동일 토큰·브랜드 가이드 사용
- **구글 시트**: 서비스 어카운트 키로 매일 22:00 push (SHEET_ID 환경변수)
- **Discord `#avoha-ops`**: 웹훅으로 신규 메시지·위기·장애 알림

## 감정 분류 가이드라인 (3 카테고리 → 10 감정, PRD v1.1)
**2-stage 판정**: 카테고리 먼저 → 세부 감정.

| 카테고리 | 감정 | 대표 패턴 |
|---|---|---|
| **평온** | 무탈 | "그냥 그런 하루", "별일 없었어", "무난" |
| **평온** | 평온 | "조용히", "차분하게", "한숨 돌렸어" |
| **행복** | 뿌듯 | "해냈어", "드디어 끝", "나 잘했지?" |
| **행복** | 기쁨 | "너무 좋아", "행복", 😊 이모티콘 다수 |
| **행복** | 만족 | "꽤 괜찮네", "충분해", "나쁘지 않아" |
| **행복** | 설렘 | 기다림·주말·약속, "두근", "기대돼" |
| **부정** | 슬픔 | "눈물", "서러워", "울컥" |
| **부정** | 짜증 | "빡쳐", "왜 이래", "짜증나" |
| **부정** | 후회 | "~할걸", "괜히", "차라리" |
| **부정** | 위로 | "지쳤어", "힘들어", 어리광 어조 |

**판정 팁**: 애매하면 유저에게 `[세부 감정 선택]` 퀵 버튼으로 넘김 (PRD F-02.1).
상세: [`guidelines/operator-classification.md`](guidelines/operator-classification.md)

## 응답 템플릿 (닥토 공방 페르소나)

### 페르소나 원칙
- 톤: 따뜻한 장인. 과한 공감("아이고", "힘내세요") 금지
- 1인칭: "저희" 또는 "닥토". 유저 호칭: "당신"
- 시그너처 동사: **깎다 · 채집하다 · 가공하다 · 세공하다 · 새겨두다**
- 판단 어려울 때: "이 감정은 새롭네요! 어떤 원석으로 깎아볼까요?" 로 유저 선택지로 넘김

## 첫 실행
```bash
# 콘솔
cd 2_avoha/ops/console
npm install
npm run dev                     # http://localhost:5174

# 스크립트 (원격 DB 대상)
cd 2_avoha/ops
cp .env.example .env            # DATABASE_URL, SHEET_ID, GOOGLE_SA_KEY
npx tsx scripts/seed-emotions.ts
```
