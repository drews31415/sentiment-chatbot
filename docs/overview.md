# 소확행 서비스 개요

## 비전

일상 속 작은 행복을 사진 한 장으로 기록하고, AI가 감정을 읽어 따뜻한 한마디를 건네는 지도 기반 감정 아카이브.

**핵심 원칙**: 한두 번의 행동만으로 감정 기록이 완료되는 초경량 경험 (최대 4단계)

## 용어 사전

| 용어 | 설명 |
|------|------|
| 소확행 기록 | image_url, created_at, latitude, longitude, raw_user_text, vision_tags, emotion_tags, context_keywords, ai_comment, skip_flag, user_id를 포함하는 기록 단위 |
| 기록 핀 | 지도 위 시각적 마커 (사진 썸네일 + 감정 키워드 + 짧은 텍스트) |
| FAB 버튼 | 홈 화면 하단 중앙 플로팅 액션 버튼 (사진 촬영/업로드 시작) |
| 최근 기록 카드 | 홈 화면 상단 가로 스와이프 최근 3~4개 카드 |
| Vision API | 사진에서 객체, 색감, 분위기, 장소 맥락 추출하는 이미지 분석 API (Gemini) |
| 감정 태그 | 감정 분류 레이블 (편안함, 설렘, 뿌듯함, 안정감, 해방감 등) |
| 감정 극성 | 감정 방향성 (긍정 / 중립 / 부정) |
| 컨텍스트 키워드 | 상황 맥락 키워드 (산책, 날씨, 휴식, 친구, 커피, 음악 등) |
| 취향 벡터 | 이미지+텍스트 통합 분석 기반 사용자 취향의 수치적 표현 |
| 대화 상태 관리자 (FSM) | 업로드→질문→응답→분석→코멘트→저장 흐름을 관리하는 상태 머신 |
| EXIF 메타데이터 | 사진 파일 내 촬영 일시, GPS 좌표 등 부가 정보 |

## 유저 스토리 요약

### 1. 지도 기반 홈 화면 (Req 1)
> 나의 소확행 기록을 지도 위에서 한눈에 확인하고 싶다.
- 지도 메인 + 기록 핀 표시
- 핀 탭 → 상세 보기 (바텀 시트)
- 상단 최근 3~4개 카드 가로 스와이프
- 하단 중앙 FAB 버튼 (가장 두드러지는 요소)

### 2. 사진 촬영 및 업로드 (Req 2)
> 카메라/갤러리로 빠르게 기록을 시작하고 싶다.
- FAB → 카메라/갤러리 선택
- 촬영/선택 완료 → **편집 단계 없이** 자동 전환
- EXIF에서 촬영 일시 + GPS 좌표 추출

### 3. AI 챗봇 인터랙션 (Req 3)
> 사진 업로드 후 AI가 먼저 말을 걸어주길 원한다.
- Vision API로 사진 분석 → 공감형 질문 1회 자동 생성
- 텍스트 답변 입력 → 코멘트 생성 단계로
- 건너뛰기 → skip_flag=true, 사진+메타만 저장
- **10초 무입력 시 건너뛰기 버튼 시각적 강조**

### 4. 이미지 분석 및 감정 추출 (Req 4)
> 사진과 텍스트로 감정/취향이 자동 분석되길 원한다.
- Vision API → vision_tags (객체, 색감, 분위기, 장소)
- 텍스트+사진 통합 → 감정 극성 + 감정 태그 + 컨텍스트 키워드
- 텍스트 없어도 사진만으로 분석 가능
- 사용자별 취향 벡터 생성/갱신
- Vision API 실패 시 → 사진+텍스트만 저장 대체 경로

### 5. AI 코멘트 생성 (Req 5)
> 기록에 대해 AI가 짧고 따뜻한 코멘트를 주길 원한다.
- **1~2문장** 짧은 감성 피드백
- 따뜻하고 자연스러운 톤 (상담/치유 어조 금지)
- 인사이트 화면에 카드 형태로 표시
- 코멘트 확인 → 기록 저장 → 홈 복귀
- 생성 실패 시 기본 감성 메시지 대체

### 6. 기록 저장 (Req 6)
> 기록이 안전하게 저장되어 다시 확인할 수 있길 원한다.
- 저장 즉시 지도 핀 반영
- 상세 조회: 사진, 일시, 위치, 텍스트, 감정 태그, AI 코멘트
- 누적 분석: 행복 트리거, 선호 장소, 선호 시간대, 감정 빈도

### 7. 대화 상태 관리 (Req 7)
> 업로드부터 코멘트까지 끊김 없이 자연스럽게 진행되길 원한다.
- FSM 기반 순차 관리 (17개 상태)
- 오류 시 → 수집 데이터 임시 저장 + 오류 안내 + 홈 복귀 옵션

### 8. UX 경량성 원칙 (Req 8)
> 모든 주요 행동을 1~2단계 안에 완료하고 싶다.
- 최소 플로우 4단계 이내 (홈 → 촬영 → 저장/답변 → 코멘트 확인)
- 사진만으로 기록 완성 가능 (텍스트 선택)
- 인사이트 = 1~2문장 짧은 피드백 (긴 나열 금지)
- 지도 중심 레이아웃 (피드형 아님)

### 9. 감정 데이터 직렬화 (Req 9)
> 감정 분석 데이터를 안정적으로 저장/복원해야 한다.
- JSON 직렬화/역직렬화 라운드트립 보장
- 유효하지 않은 JSON → 구체적 파싱 오류 메시지

## 정보 아키텍처

```
/ (Home - 지도 + 최근 기록 카드)
│   └── Record Detail (핀 탭 → 바텀 시트 오버레이)
├── /capture (FAB → 풀스크린 플로우, 하단 네비 숨김)
│   ├── Step 1: 사진 (카메라/갤러리)
│   ├── Step 2: 채팅 (AI 질문 + 응답)
│   └── Step 3: 인사이트 (AI 코멘트 + 저장)
│   └── → 홈 복귀
├── /discovery (주간 요약 - 두 번째 탭)
└── /profile (설정, 기록 히스토리 - 세 번째 탭)
```
